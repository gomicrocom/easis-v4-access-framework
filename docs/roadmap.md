# Roadmap

## Phase 1

- establish repository structure for exported Access objects and documentation
- define architecture conventions for tenant isolation and framework layering
- formalize INI-based configuration approach
- define multilingual support strategy
- define licensing boundaries for core and optional modules

## Phase 2

- introduce framework bootstrap components in VBA
- add configuration loading and environment resolution
- add translation service foundation
- add license evaluation service
- define first backend schema conventions

## Phase 3

- export and organize Access objects into the repository structure
- implement module registration and activation flow
- add first optional module integrations
- introduce test and validation tooling for exports and framework checks
- implement tag-driven validation system
- introduce translation service (T / TEx)
- add Tag Composer UI
- implement validation highlighting UX

## Phase 4

- refine deployment workflow for frontend and per-tenant backends
- add migration/versioning strategy for backend databases
- expand module catalog and operational documentation

## Phase 5

- implement document output service (PDF)
- introduce output path management
- implement email service (CDO)
- build central batch handler
- integrate CAMT.054 processing
- add scan integration (NAPS2)
